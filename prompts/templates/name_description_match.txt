**CONTEXT:** The data entry team is prone to errors. They often:

3. Leave descriptions empty.

**TASK: Detect FRAUD and NON-ASSET entries. Do not be a spell-checker. Do not be a pedant.**

**THE "UNIVERSAL VALIDATION" RULES (apply in order):**

0. **HIGHEST PRIORITY - SERVICE/EXPENSE/CHARGE DETECTION = ALWAYS REJECT**
 - BEFORE any other check, determine if the asset name represents a PHYSICAL FIXED ASSET or a SERVICE/EXPENSE/CHARGE entry.
 - A physical fixed asset is a TANGIBLE OBJECT you can touch: laptop, chair, AC unit, printer, vehicle, machine, etc.
 - A service/expense/charge is NOT a physical asset. It is a cost, fee, or labor entry.
 - If the asset name contains ANY of these keywords (alone or combined with an asset name), it is NOT a valid asset -> **REJECT immediately**:
 "installation charges", "installation cost", "service charges", "service cost", "labor charges", "labor cost",
 "maintenance charges", "maintenance cost", "repair charges", "repair cost", "AMC charges", "AMC cost",
 "shipping charges", "freight charges", "handling charges", "delivery charges", "setup charges", "setup cost",
 "commission", "consultancy", "consulting fees", "warranty charges", "insurance charges", "rental charges"
 - CRITICAL: Even if an asset name is PART of the entry (e.g., "AC Installation Charges", "Laptop Repair Cost"), the ENTIRE entry is still a service/expense -> **REJECT**.
 - The key test: "Can you physically tag this with a barcode sticker?" If NO -> REJECT.

 Examples of NON-ASSETS (REJECT all):
 - "AC Installation Charges" -> REJECT (installation service, not an AC unit)
 - "Laptop Installation Charges" -> REJECT (service charge, not a laptop)
 - "Software Installation Cost" -> REJECT (service cost)
 - "Repair Charges" -> REJECT (service)
 - "Maintenance Charges" -> REJECT (service)
 - "Service Charges" -> REJECT (service)
 - "Labor Charges" -> REJECT (service)
 - "AMC Charges for Printer" -> REJECT (annual maintenance contract, not a printer)
 - "Freight and Handling Charges" -> REJECT (logistics cost)

 Output for rejected non-assets:

 reasoning: "User Claim: SERVICE/EXPENSE ([entry name]). This is not a physical fixed asset - it is a service/expense/charge entry that cannot be physically tagged or verified."

0.5 **ASSET TYPE MUST MATCH (CRITICAL - CHECK NAME AND DESCRIPTION SEPARATELY)**
 - The IMAGE / Previous Analysis is the ground truth for what the physical asset actually is.
 - Extract the asset TYPE from the User's Asset Name (laptop, printer, chair, AC, monitor, phone, etc.).
 - Extract the asset TYPE from the User's Description when provided.
 - **REJECT** if the asset NAME type contradicts the image, even when the brand matches.
 - **REJECT** if the asset NAME and DESCRIPTION specify different asset types (e.g., name says Laptop, description says Printer).
 - Brand match alone is NOT enough. "HP Laptop" vs image of HP Printer -> **REJECT**.
 - Do NOT approve because the description mentions the correct type while the asset name is wrong. Both fields matter; wrong asset name = **REJECT**.

 Examples (REJECT):
 - Name: "HP Laptop", Description: "HP Black and White Printer", Image: HP LaserJet printer -> **REJECT** (name says Laptop, image shows Printer; name and description also conflict)
 - Name: "Dell Monitor", Image: Laptop -> **REJECT**

 Examples (APPROVE):

 - Name: "ThinkPad", Image: Lenovo laptop (no brand in name, image matches laptop type) -> **APPROVE**
 - Name: "Chair", Image: Office chair -> **APPROVE**

1. **GENERIC NAMES = APPROVE ONLY IF IMAGE MATCHES TYPE (Physical Assets Only)**
 - If the user describes a *physical asset Category* (e.g., "Chair", "Pump", "Laptop", "Printer") without a brand, **APPROVE only if the image shows that same category**.
 - Trust generic names ONLY when the image confirms the same object type (e.g., User: "Chair", Image: Chair).
 - **REJECT** when generic name and image types differ (e.g., User: "Laptop", Image: Printer).

2. **MODEL NAMES & BRANDS = "INNOCENT UNTIL PROVEN GUILTY"**
 - **Model Name Handling:** If user enters a Model Name (e.g., "Latitude 5420"), check if the image matches that model's parent brand (Dell). If yes, **APPROVE**.
 - **Model Name Only (The "Duffer" Rule):** Data entry often puts ONLY the model (e.g., "Mustang", "JCB 3DX", "ThinkPad") and omits the Brand. This is VALID. Do not reject because the Brand is missing.
 - **Scenario A (Match):** User says "Brand X" or "Model X", and you see that Brand/Model -> **APPROVE**.
 - **Scenario B (Invisible/Niche):** User says "Kirloskar Pump". You see a Pump, but NO visible logo (or logo is obscure/dirty). **APPROVE**. (Give benefit of the doubt).
 - **Scenario C (Brand Contradiction):** User says "Dell". You CLEARLY see an "HP" logo. **REJECT**.
 - **Scenario D (Asset Type Contradiction):** User name says "Laptop" but image clearly shows a Printer (or vice versa). **REJECT** even if brand matches.

3. **IGNORE DISTRACTORS**
 - Do not confuse these with manufacturers:
 - Component brands (Intel, Shimano, Gore-Tex, Windows)
 - Certifications (UL, CE, ISO, Energy Star)
 - Safety/Warning labels

4. **ALLOW TYPOS & PHONETICS**
 - "Lenevo" == "Lenovo"
 - "Herman Miler" == "Herman Miller"
